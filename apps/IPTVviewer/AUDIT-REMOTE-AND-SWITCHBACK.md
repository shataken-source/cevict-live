## 2. What’s Not Finished

### 2.1 WebSocket server (LAN / Network mode)

- **Remote** offers "LAN / Network": user enters TV IP, remote connects to `ws://IP:8765`.
- **No WebSocket server exists** in this repo that listens on port **8765**:
  - Not in IPTVviewer (no Node/Express WS server).
  - Not in SwitchbackSimple (Android WebView has no WS server).
  - Not in `switchback.html` / `switchback-simple.html` (browser; no WS server).
- **Result:** "LAN / Network" always fails (connection refused). Only **Same Device** (localStorage) works, and only when remote and TV app share the same origin (e.g. same browser, different tabs).

**To finish LAN mode:** Add a small WebSocket server that:
- Listens on 8765 (e.g. on the machine/device running Switchback TV).
- On message `{ action, ... }`, runs the same logic as `handleRemoteCommand` / `handleRemoteCmd`.
- Optionally pushes state (same as `sb_state`) to the phone over the same socket.

### 2.2 Same-tab polling (SwitchbackSimple)

- Comment in `app.js` says: *"We listen for storage events (cross-tab) and also poll (same-tab fallback)."*
- Only the **storage** listener is implemented; there is **no** `setInterval` polling for `sb_remote_cmd`.
- **Impact:** If the TV app is the only tab (e.g. Android WebView single page), storage events from another tab don’t exist; on the same device, same-tab polling would be needed. For **phone → TV on another device**, storage is irrelevant; WebSocket is required (see above).

### 2.3 Universal remote

- **Intent:** Remote was intended to be "for the switchback.tv app but also a universal remote."
- **Current state:** The UI and code are **only** Switchback-specific:
  - Switchback slots (SB 1–4), sb_cycle, sb_jump.
  - Navigation and playback match Switchback/IPTV (Guide, DVR, Live TV, etc.).
  - No section for other devices (e.g. Roku, Fire TV, generic IR/HDMI-CEC, or other apps).
- **Conclusion:** The **universal remote** part was never implemented; only the Switchback TV remote was built.

## 3. IPTVviewer vs Switchback

| | IPTVviewer | Switchback |
|--|------------|------------|
| **IPTVviewer** | React Native app (Android TV); also has **website** (HTML/JS) that plays streams and handles remote. | N/A (product name / feature set). |
| **Switchback TV** | Refers to the **multi-slot live TV** experience (switch between 4 channel slots). | Implemented in **SwitchbackSimple** (Android WebView app) and in **IPTVviewer website** (index.html). |
| **switchback.html** | Standalone HTML app (different from IPTVviewer website). | Full Switchback-style UI; no remote handler in the snippet audited. |
| **switchback-simple.html** | Simpler HTML version. | Config/setup style; not the main player. |
| **SwitchbackSimple** | Android app in `apps/SwitchbackSimple/` (WebView + assets). | Main Android TV app: `app.js` has remote listener + `publishTVState`. |

So: **IPTVviewer** = brand/app family; **Switchback** = the 4-slot live TV UX. The **phone remote** is the "Switchback TV Remote" and targets both the IPTVviewer website and the SwitchbackSimple app.

## 5. Recommendations

1. **Finish LAN remote:** Implement a WebSocket server on 8765 (e.g. in a small Node or in the Android app via a native module / companion process) that receives the same `{ action, ... }` payloads and calls the same handler as `handleRemoteCommand` / `handleRemoteCmd`.
2. **Optional – same-tab fallback:** If you need remote to work with TV app in a single WebView (same tab, no second tab), add a `setInterval` in the TV app that reads `localStorage.getItem('sb_remote_cmd')`, processes it, then removes it (same as current cleanup in remote.js).
3. **Universal remote:** Treat as a new feature: extra section or mode in the remote (or a separate entry point) for other devices (e.g. Roku/Fire TV APIs, or IR/HDMI-CEC if hardware exists), with device discovery and a second command set.

---

**Bottom line:** The **Switchback TV Remote** is implemented for **same-device (localStorage)** and **LAN (WebSocket)**. Same-tab polling is in place in SwitchbackSimple. The **universal remote** (other devices/apps) is still not implemented.
